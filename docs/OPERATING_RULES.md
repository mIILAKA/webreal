# Operating Rules

## ブランチとPR

- `main` への直接pushは禁止。
- すべての変更はPRで提出する。
- PRは小さく保ち、目的が1つに見える粒度にする。
- PR本文には、変更目的、変更ファイル、検証結果、公式確認の有無を書く。

## 公式情報の確認

以下を扱う場合は公式ソースを確認する。

- 料金
- 利用制限
- プラン名
- API仕様
- 利用規約
- 商標、ブランド名、サービス名
- 法律、税務、医療、金融など高リスク情報

確認した場合は、記事またはPRに「確認日」と「公式URL」を残す。

## 実装ルール

- 既存のNext.js構成、TypeScript、CSS方針に合わせる。
- 不要なライブラリを追加しない。
- 変更後は必ず `npm run lint` と `npm run build` を実行する。
- 失敗したチェックを放置しない。

## コンテンツ更新ルール

- 検索意図を明確にする。
- 内部リンクを追加する。
- FAQを追加または更新する。
- 最終更新日を表示またはデータとして管理する。
- 料金・制限・仕様は「変更される可能性がある」と明記する。

## データを見る順番

1. Google Search Console
2. Google Analytics
3. 実際の検索結果
4. 公式ドキュメント
5. 競合ページの構成

競合ページは参考にとどめ、文章や表現をコピーしない。
