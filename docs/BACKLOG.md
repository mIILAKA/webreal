# Backlog

AI Founder自動運営システムで扱う改善候補です。PR化する前に、目的、根拠、検証方法を明確にします。

## 優先度A

- [x] 記事データに最終更新日と公式確認日を追加する
  - 根拠: 料金・制限情報の信頼性を高め、公開直後から法務・SEO・UXの土台を整えるため
  - 検証: 各記事に検索意図、最終更新日、公式情報確認日、公式情報リンクが表示される

- [ ] Analytics連携方法を決める
  - 根拠: 収益化前にページ行動を見るため
  - 検証: ページビュー、流入元、エンゲージメントを確認できる

- [ ] Search Console連携方法を決める
  - 根拠: 週次レビューを実データで行うため
  - 検証: クリック数、表示回数、CTR、平均掲載順位を取得できる

## 優先度B

- [ ] 既存記事の内部リンク表を作る
  - 根拠: 回遊とSEO評価を高めるため
  - 検証: 各記事に関連リンクがある

- [ ] 価格・制限情報の公式URL一覧を作る
  - 根拠: 更新時の確認漏れを減らすため
  - 検証: 各サービスの公式URLと確認日が記録される

- [ ] FAQ構造化データの導入を検討する
  - 根拠: 検索結果での理解を助けるため
  - 検証: build成功、構造化データテストで問題なし

## 優先度C

- [ ] 週次レビュー結果をIssue化するGitHub Actionsを追加する
  - 根拠: 改善案を蓄積するため
  - 検証: 手動実行でIssueが作成される

- [ ] AIによるPR下書き作成を検討する
  - 根拠: 改善速度を上げるため
  - 検証: main直pushなし、PRのみで提案される
