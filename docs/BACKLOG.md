# Backlog

AI Founder自動運営システムで扱う改善候補です。PR化する前に、目的、根拠、検証方法を明確にします。

## 運用ルール

- 1PR = 1目的にする。
- Analytics、SEO、Revenue、Content、Automationは別ブランチで作業する。
- PRサイズは原則300行以内にする。
- 大規模変更は複数PRに分割する。
- 改善候補は該当カテゴリに置き、複数カテゴリにまたがる場合は分割して記録する。
