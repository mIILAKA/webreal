# Backlog

AI Founder自動運営システムで扱う改善候補です。PR化する前に、目的、根拠、検証方法を明確にします。

## SEO

- [ ] 既存記事の内部リンク表を作る
  - 根拠: 回遊とSEO評価を高めるため
  - 検証: 各記事に関連リンクがある

- [ ] FAQ構造化データの導入を検討する
  - 根拠: 検索結果での理解を助けるため
  - 検証: build成功、構造化データテストで問題なし

- [ ] Search Console連携方法を決める
  - 根拠: 週次レビューを実データで行うため
  - 検証: クリック数、表示回数、CTR、平均掲載順位を取得できる

## Content

- [ ] 価格・制限情報の公式URL一覧を作る
  - 根拠: 更新時の確認漏れを減らすため
  - 検証: 各サービスの公式URLと確認日が記録される

## Product

- [ ] 診断実行数の計測方法を決める
  - 根拠: 記事閲覧から診断利用につながっているかを見るため
  - 検証: 診断実行イベント数を週次レポートに記録できる

## Analytics

- [ ] Analytics連携方法を決める
  - 根拠: 収益化前にページ行動を見るため
  - 検証: ページビュー、流入元、エンゲージメントを確認できる

- [ ] 週次レビュー結果をIssue化するGitHub Actionsを追加する
  - 根拠: 改善案を蓄積するため
  - 検証: 手動実行でIssueが作成される

- [ ] AIによるPR下書き作成を検討する
  - 根拠: 改善速度を上げるため
  - 検証: main直pushなし、PRのみで提案される

## Revenue

- [ ] 収益化導線を置く候補ページを決める
  - 根拠: 実データで人気記事と検索流入を確認してから収益化するため
  - 検証: 候補ページ、根拠データ、配置案が週次レポートに記録される
