# Weekly Review

## 目的

AIが毎週、Search Consoleで見るべき項目と次の改善案を出すためのレビュー手順です。v1では自動PR作成は行わず、改善案の生成までを想定します。

## 実行タイミング

- 毎週1回
- GitHub Actionsの手動実行
- 必要に応じてスケジュール実行

## Search Consoleで見る項目

- 合計クリック数
- 合計表示回数
- 平均CTR
- 平均掲載順位
- クエリ別クリック数
- クエリ別表示回数
- ページ別クリック数
- ページ別表示回数
- 掲載順位が11から30位のクエリ
- 表示回数が多いがCTRが低いクエリ
- 直近7日と過去28日の変化

## Analyticsで見る項目

- ページビュー
- エンゲージメント
- 流入元
- 離脱が多いページ
- 回遊されているページ
- 収益化導線を置く候補ページ

## 週次レビュー出力テンプレート

```text
レビュー日:
対象期間:

1. 今週の変化
- クリック数:
- 表示回数:
- CTR:
- 平均掲載順位:

2. 注目クエリ
- 伸びたクエリ:
- CTRが低いクエリ:
- 11から30位のクエリ:

3. 改善候補ページ
- ページ:
- 理由:
- 改善案:

4. 次にやること
- 優先度A:
- 優先度B:
- 優先度C:

5. 公式確認が必要な項目
- 料金:
- 制限:
- 規約:

6. PR化する場合の変更案
- ブランチ名:
- 変更ファイル:
- 検証コマンド:
```

## 優先順位の考え方

優先度A:
表示回数が多く、順位が11から20位で、少ない追記で改善できるもの。

優先度B:
CTRが低く、タイトルやメタディスクリプションで改善できるもの。

優先度C:
新規記事、構造変更、大きな調査が必要なもの。
