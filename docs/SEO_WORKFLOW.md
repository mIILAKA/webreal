# SEO Workflow

## 目的

検索意図に合う記事を継続的に改善し、クリック数、CTR、平均掲載順位、回遊率を高める。

## 新規記事の作成手順

1. 検索意図を1文で定義する。
2. 主キーワードと関連キーワードを決める。
3. 読者が比較したい軸を整理する。
4. 公式情報を確認する。
5. タイトル、メタディスクリプション、見出し、FAQを作る。
6. 既存記事から内部リンクを張る。
7. 記事内から関連ページへ内部リンクを張る。
8. 最終更新日と公式確認日を入れる。
9. `npm run lint` と `npm run build` を実行する。

## 既存記事の改善手順

Search Consoleで以下を見る。

- 表示回数が多いがCTRが低いクエリ
- 平均掲載順位が11から30位のクエリ
- クリックされているが滞在や回遊が弱いページ
- 想定外のクエリで表示されているページ

改善案の例:

- タイトルを検索意図に寄せる
- 冒頭に結論を追加する
- 比較表や判断基準を追加する
- FAQを追加する
- 関連記事への内部リンクを追加する
- 古い価格・制限情報を公式情報で更新する
