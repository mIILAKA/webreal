# AI Founder自動運営システム v1

## 目的

`webreal` を、AIが半自動で継続改善できる状態にする。AIは分析、改善案の作成、記事更新案、SEO改善案、品質チェックを支援する。ただし、本番反映と収益化判断は人間の確認を必須とする。

## 対象範囲

- SEO記事の追加・改善
- 内部リンク改善
- FAQ追加
- メタデータ改善
- Search ConsoleとAnalyticsを使った週次改善案の作成
- 既存ページの品質改善
- lint/buildによる品質確認

## AIができること

- 調査メモ、改善案、PR本文、記事下書きを作成する。
- 既存コードと文体に合わせてページや記事を更新する。
- `docs/WEEKLY_REVIEW.md` に沿って週次レビューの観点を整理する。
- `npm run lint` と `npm run build` を実行し、結果を報告する。

## AIがしてはいけないこと

- `main` へ直接pushする。
- 公式確認なしに料金、制限、仕様、規約情報を断定する。
- 他サイトの記事、画像、表、ランキングを無断転載する。
- 収益化リンク、広告、計測タグを人間の承認なしに追加する。
- 医療、法律、金融など高リスク領域で専門家の判断に見える断定をする。

## 運用フロー

1. データ確認: Search Console、Analytics、既存順位、流入、クリック率を確認する。
2. 課題抽出: 表示回数が多いがCTRが低いクエリ、順位が11から30位の記事、内部リンク不足を探す。
3. 改善案作成: 検索意図、見出し、FAQ、内部リンク、公式情報確認が含まれる案を作る。
4. ブランチ作成: `ai/YYYY-MM-DD-topic` のようなブランチ名を使う。
5. 実装: 変更を小さく分ける。
6. 検証: `npm run lint` と `npm run build` を実行する。
7. PR作成: 目的、変更内容、確認結果、リスクを記載する。
8. 人間レビュー: 内容、法務、収益化、公式情報を確認してからマージする。

## 成功指標

- Search Consoleのクリック数、表示回数、CTR、平均掲載順位
- Analyticsのページビュー、エンゲージメント、流入元
- 記事の更新頻度
- 内部リンクの網羅性
- 収益化前の意図一致と信頼性

## v1の制限

- 週次レビューは雛形のみ。自動でAI PRを作成しない。
- Search ConsoleとAnalytics API連携は未設定。
- 実データ取得は人間または将来のAPI連携で行う。
