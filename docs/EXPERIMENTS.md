# Experiments

## 目的

改善施策を仮説、実施日、結果、学びで記録し、AIが思いつきで改善を繰り返さない状態を作ります。

## 記録ルール

- 実施前に仮説と検証方法を書く。
- 実施後に結果と学びを書く。
- 結果が出ていない施策は、次の施策の根拠にしない。
- 料金、制限、仕様、規約に関わる施策は、公式確認日とURLを残す。

## 実験テンプレート

```text
## YYYY-MM-DD 実験名

仮説:

実施日:

対象ページ:

変更内容:

参照データ:

検証方法:

結果:

学び:

次の対応:
```
